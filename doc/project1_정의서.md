# 프로젝트 1 - 시스템 정의서

## 1. 프로젝트 명칭

DevTrack 개발 프로젝트

## 2. 개발 시스템 명칭

소프트웨어 개발 프로젝트 통합 관리 시스템 (DevTrack)

## 3. 개발 시스템 설명

소프트웨어 개발 조직에서는 요구사항 수집, 일정 관리, 팀원 역할 분담, 산출물 관리 등의 작업이
여러 도구에 분산되어 처리되는 경우가 많다.
이로 인해 요구사항 변경 이력을 추적하기 어렵고, 개발 일정과 실제 진행 상황 간의 괴리가 발생하며,
팀원 간 업무 현황 공유가 비효율적으로 이루어진다.
본 시스템은 요구사항 관리, 일정 및 진행 상황 추적, 산출물 버전 관리, 팀원 역할 및 권한 관리를
하나의 플랫폼에서 통합 제공하는 것을 목표로 한다.
이를 통해 개발 조직은 프로젝트 전반을 체계적으로 관리하고, 투명한 협업 환경을 구축할 수 있다.

## 4. 핵심 기능

1. **요구사항 등록 및 추적** - 고객 요구사항을 등록하고 변경 이력을 추적하여 최신 요구사항을 항상 파악 가능
2. **일정 및 마일스톤 관리** - 프로젝트 전체 일정과 세부 마일스톤을 등록하고 진행 상황을 시각적으로 확인 가능
3. **팀원 역할 및 권한 관리** - 팀원별 역할을 지정하고 접근 권한을 설정하여 보안성과 책임 소재를 명확히 함
4. **산출물 버전 관리** - 문서, 설계서 등 산출물의 수정 이력을 보관하고 이전 버전으로 복원 가능
5. **진행 상황 대시보드** - 전체 프로젝트의 진행률, 완료된 태스크, 남은 일정을 한눈에 파악 가능
6. **알림 및 공지 기능** - 일정 변경, 요구사항 업데이트 등 주요 변경 사항 발생 시 팀원에게 자동 알림 제공

## 5. 유사 소프트웨어 분석

| 소프트웨어 | 주요 기능 | 장점 | 단점 | DevTrack과 차이점 |
|-----------|----------|------|------|----------------|
| Jira | 이슈 트래킹, 스프린트 관리, 요구사항 관리 | 강력한 기능, 다양한 플러그인 지원 | 설정이 복잡하고 소규모 팀에 과도한 기능 | DevTrack은 요구사항 변경 이력과 산출물 관리를 통합 제공 |
| Trello | 칸반 보드, 태스크 관리 | 직관적인 UI, 빠른 설정 | 요구사항 추적 및 버전 관리 기능 부족 | DevTrack은 요구사항 추적과 산출물 버전 관리까지 지원 |
| Notion | 문서 작성, 프로젝트 관리 | 자유로운 구성, 다양한 템플릿 | 대규모 프로젝트 일정 관리 기능 부족 | DevTrack은 일정 관리와 진행 상황 추적에 특화 |
| Confluence | 문서 관리, 팀 협업 | 문서화 기능 강력, Jira 연동 | 단독 사용 시 프로젝트 관리 기능 부족 | DevTrack은 요구사항부터 산출물까지 하나의 플랫폼에서 관리 |
